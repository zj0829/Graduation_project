三、系统启动（命令行方式）

3.1 启动 Docker Desktop

手动双击桌面或开始菜单中的 Docker Desktop 图标，等待其状态变为 Running

# 验证Docker是否就绪
docker info

3.2 创建Docker网络

docker network create pentest-network

3.3 启动安全测试工具容器

# Nmap - 端口扫描
docker run -d --name pentest-nmap --network pentest-network --entrypoint="" instrumentisto/nmap:latest sleep infinity

# Amass - 子域名枚举
docker run -d --name pentest-amass --network pentest-network --entrypoint="" caffix/amass:latest sleep infinity

# SQLmap - SQL注入检测（需先构建镜像）
cd tools\sqlmap
docker build -t pentest-sqlmap .
docker run -d --name pentest-sqlmap --network pentest-network --entrypoint="" pentest-sqlmap:latest sleep infinity

# OWASP ZAP - Web漏洞扫描
docker run -d --name pentest-owasp-zap --network pentest-network -p 8090:8090 -e TZ=Asia/Shanghai zaproxy/zap-stable:latest zap.sh -daemon -port 8090 -host 0.0.0.0 -config api.addrs.addr.name=.* -config api.addrs.addr.regex=true -config api.key=

# PostgreSQL - Metasploit数据库
docker run -d --name pentest-metasploit-db --network pentest-network -e POSTGRES_USER=msf -e POSTGRES_PASSWORD=msf_2026_secure -e POSTGRES_DB=msf postgres:15-alpine

# Metasploit - 渗透测试框架
docker run -d --name pentest-metasploit --network pentest-network -p 55553:55553 -p 4444:4444 -e MSF_DATABASE_CONFIG=postgres://msf:msf_2026_secure@pentest-metasploit-db:5432/msf --entrypoint="" metasploitframework/metasploit-framework:latest sleep infinity
验证容器状态：
docker ps — 应看到6个容器全部为 Up 状态

3.4 启动MCP服务

# 进入MCP服务目录
cd mcp-server

# 首次运行需安装依赖
npm install

# 启动MCP服务（端口9876）
node server.js
验证MCP服务：
浏览器访问 http://localhost:9876/health，应返回 {"status": "healthy"}

3.5 启动后端API服务

# 进入项目根目录
cd d:\Graduation_project\Test

# 首次运行需安装Python依赖
pip install -r requirements.txt

# 启动后端服务（端口8000）
python main.py
验证后端服务：
浏览器访问 http://localhost:8000/docs，应看到FastAPI自动文档页面

3.6 打开前端页面

# 浏览器直接访问
start http://localhost:8000/static/index.html

四、一键启动方式

双击 start.bat 即可自动完成上述所有步骤

脚本会自动：检查Docker → 创建网络 → 启动容器 → 启动MCP → 启动后端 → 打开浏览器

双击 stop.bat 停止所有服务

脚本会自动：停止后端进程 → 停止MCP进程 → 停止Docker容器

五、使用流程

1
输入测试目标
在前端页面的"测试目标"输入框中填入目标网站域名或IP地址，例如：testphp.vulnweb.com
2
填写测试需求
描述你的测试需求，例如："对该网站进行全面安全扫描，检测开放端口和SQL注入漏洞"
3
选择工具策略
智能推荐 MCP自动选择最合适的工具组合
全面扫描 使用所有可用工具
快速扫描 仅使用Nmap快速探测
自定义 手动选择要使用的工具
4
点击"开始测试"
系统将自动执行：MCP连接 → 工具调用 → 结果分析 → 报告生成
5
查看报告
测试完成后，报告以Markdown格式展示在页面中，可直接阅读或复制保存

十、常见问题

Q: 启动后端报错 "Connection refused" 连接MCP失败？

A: 确保MCP服务已启动（端口9876）。检查命令：netstat -ano | findstr ":9876"。如果MCP未运行，先执行 cd mcp-server && node server.js

Q: Docker容器启动失败？

A: 确保Docker Desktop已启动。如果端口被占用，先删除旧容器：docker rm -f 容器名，然后重新创建。

Q: 测试超时？

A: 真实网站扫描需要较长时间（Nmap约2-5分钟，SQLmap约5-10分钟）。建议先使用"快速扫描"模式测试连通性。

Q: DeepSeek API报错？

A: 检查 .env 文件中的 DEEPSEEK_API_KEY 是否正确。确保API账户有余额。

Q: 如何配置CherryStudio连接MCP？

A: 在CherryStudio的MCP服务器配置中，添加地址 http://localhost:9876，协议选择SSE。

十一、数据流

用户输入 → 前端页面(8000)
    ↓ POST /api/orchestrator/execute-test
后端routes.py
    ↓ httpx → 127.0.0.1:9876/jsonrpc
MCP服务
    ↓ initialize → tools/list → tools/call
Docker容器执行工具（nmap/sqlmap/amass/zap/metasploit）
    ↓ 返回工具执行结果
后端routes.py
    ↓ llm_interface.analyze_results()
DeepSeek API → 分析漏洞、评估风险
    ↓ llm_interface.generate_report()
DeepSeek API → 生成Markdown报告
    ↓ 返回test_id
前端页面 → GET /api/orchestrator/test-result/{test_id}
    ↓ 渲染Markdown报告
